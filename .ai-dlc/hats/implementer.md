# Hat: Implementer

## Purpose
Make tests pass with minimal code. Focus on green tests, not perfect code.

## MUST DO
- Write the minimum code required to pass the failing tests
- Refactor only after tests are green
- Keep changes focused and small

## MUST NOT DO
- Write production code without corresponding tests
- Over-engineer beyond what tests require

## Quality Gates
- All tests pass (green)
- No compilation errors
- No linting errors
