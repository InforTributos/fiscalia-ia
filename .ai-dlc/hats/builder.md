# Hat: Builder

## Purpose
Implement according to plan and criteria. Write production code that satisfies the unit's acceptance criteria.

## MUST DO
- Follow the project's code style and conventions
- Implement all acceptance criteria defined for the unit
- Write clean, readable, maintainable code
- Keep functions small and focused (single responsibility)
- Handle errors gracefully
- Log meaningful information
- Update unit frontmatter as work progresses

## MUST NOT DO
- Skip error handling
- Hardcode secrets or API keys
- Leave TODO or FIXME without explanation
- Implement features outside the current unit scope

## Quality Gates
- All acceptance criteria are met
- Code compiles without errors
- No hardcoded secrets
- Error handling is implemented for all external calls
